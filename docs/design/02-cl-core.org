* Why CL-core first (vs roadmap's Scheme Phase 2)

The end goal that motivated this work is a self-improving agent that
stands on decades of Lisp AI heritage (genetic programming, symbolic
AI).  That heritage is overwhelmingly *Common Lisp*.  CL is also a
*Lisp-2* like Elisp, so cl-lib / eieio already carry most of the
runtime — translation is near-identity for the core.  Scheme's
Lisp-1 + hygienic macros + call/cc make it the harder, less
AI-relevant target.  So we front-load CL core; the dialect-neutral IR
means Scheme still benefits later.

Lambda-list handling: a CL λ-list needs =cl-defun=/=cl-function= iff it
contains =&key=, =&aux=, or an =&optional=/=&rest= var with a default
or supplied-p.  Plain =(a b &optional c &rest d)= → plain =defun=.

Declarations =(declare …)= and =(the TYPE expr)= are *stripped*
(=the= keeps its expr) — they are hints, semantically droppable.

* Scope — OUT (reject loudly, =ldb-cl-unsupported-form-error=)

CLOS (=defclass/defmethod/defgeneric=), conditions
(=define-condition/handler-case/restart-case=), macros
(=defmacro/macrolet/define-symbol-macro=), =loop=, multiple values
(=values/multiple-value-bind=), packages
(=in-package/defpackage=), I/O with divergent semantics
(=print/princ/write/read=), =tagbody/go=, =eval-when=,
nested backquote, =call/cc=-equivalents, binding/clause forms
(=destructuring-bind/with-slots/with-accessors=).  Each signals with
the offending form — never silent.  These land in later increments
(macros gated on the L3 hygiene decision).

Rationale: a narrow, *correct* core that runs real algorithms beats a
broad, subtly-wrong translator.  Matches CLAUDE.md invariant 4.

* IR tags

Re-used (01-overview L2): =literal= =ref= =quote= =call= =if= =let=
=lambda= =module=.

Added this phase (allowed — new =:tag= values only):

| Tag       | =:form= shape                                              |
|-----------+------------------------------------------------------------|
| =defun=   | =(:name SYM :params LL :body (NODE…) :cl BOOL)=            |
| =defvar=  | =(:name SYM :value NODE :kind defvar\vert defparameter)=   |
| =cond=    | =(:clauses ((TEST-NODE BODY-NODE…) …))=                   |
| =special= | =(:head SYM :args (NODE…))=  head-passthrough special form |
| =bind=    | =(:head SYM :spec (VAR ITER…) :body (NODE…))= dolist/dotimes|
| =locals=  | =(:head SYM :defs ((NAME LL (BODY…))…) :body (NODE…))= labels/flet|

=let= form gains =:star BOOL=.  =lambda= form gains =:cl BOOL=.

=special= is the work-horse for elisp-identical forms (when/unless/
and/or/not/progn/prog1/prog2/setq/setf/catch/throw/unwind-protect/
ignore-errors/block/return-from/function) — translate every arg as an
expression, pass the (possibly remapped) head through.  =ref= nodes
emit a bare symbol, so =setq=/=block= names round-trip correctly.

* Pipeline

#+begin_src text
  CL source string
    │ ldb-cl-read-all-from-string   (preprocess #\C→?C, |sym|, then `read')
    ▼
  Elisp S-expressions
    │ ldb-cl-form-to-ir             (dispatch on head; reject OUT-of-scope)
    ▼
  IR nodes (dialect-neutral, :origin 'cl)
    │ ldb-emit-elisp-node           (general IR → Elisp, in ldb-emit-elisp.el)
    ▼
  Elisp source forms (data) — byte-compilable, cl-lib-dependent
#+end_src

Public API (=ldb-cl.el=):

#+begin_src elisp
(ldb-cl-translate-string SOURCE)  ; → list of Elisp forms (data)
(ldb-cl-translate-form FORM)      ; → one Elisp form
(ldb-cl-translate-file PATH)      ; → list of Elisp forms
#+end_src

Output is *data* (caller pp / write / eval), same policy as Phase 1
(L5).  Emitted code requires =(require 'cl-lib)= at the consumer.

* Open questions

- OQ1. =defstruct= → =cl-defstruct= — DONE (4b).  Name/option spec is
  kept verbatim (cl-defstruct accepts the same =(:constructor ...)= etc.),
  slot defaults are translated.  Exotic =:type vector= layouts untested.
- OQ2. =format= directive translation — DONE (4b) for the basic subset
  (=~a ~s ~d ~f ~e ~g ~x ~o ~c ~% ~& ~~=, destinations =nil= / =t=).
  Parameterized (=~5d=, =~,2f=), iteration (=~{...~}=) and conditional
  (=~[...~]=) directives are rejected loudly; a later increment can add
  a real directive parser.
- OQ3. Macro translation stays gated on L3 (pre-expand vs native
  hygiene); no macros in core.
